## 环境准备

### 系统要求

- **操作系统**: Linux (推荐 Ubuntu 20.04+), Windows, macOS
- **Docker**: 20.10.0+
- **Docker Compose**: 2.0.0+
- **内存**: 至少 4GB RAM (推荐 8GB+)
- **磁盘**: 至少 20GB 可用空间

### 安装 Docker 和 Docker Compose

#### Ubuntu/Debian

```bash
# 更新包索引
sudo apt-get update

# 安装 Docker
sudo apt-get install -y docker.io

# 启动 Docker 服务
sudo systemctl start docker
sudo systemctl enable docker

# 安装 Docker Compose
sudo apt-get install -y docker-compose

# 验证安装
docker --version
docker-compose --version
```

#### CentOS/RHEL

```bash
# 安装 Docker
sudo yum install -y docker
sudo systemctl start docker
sudo systemctl enable docker

# 安装 Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装
docker --version
docker-compose --version
```

#### Windows

1. 下载并安装 [Docker Desktop](https://www.docker.com/products/docker-desktop)
2. 确保 WSL2 已启用
3. 重启系统

#### macOS

```bash
# 使用 Homebrew 安装
brew install --cask docker

# 或从官网下载 Docker Desktop
# https://www.docker.com/products/docker-desktop
```

### 配置 Docker 镜像加速 (可选，国内环境推荐)

```bash
# 编辑 Docker 配置
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "https://hub-mirror.c.163.com",
    "https://registry.docker-cn.com"
  ]
}
EOF

# 重启 Docker
sudo systemctl daemon-reload
sudo systemctl restart docker
```

## 快速部署

### 1. 获取项目代码

```bash
# 进入工作目录
cd /path/to/your/project

# 假设项目已克隆到当前目录
cd unimap-icp-hunter

# 查看项目结构
ls -la
```

### 2. 配置环境变量

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑配置文件，填入 API Keys
vim .env
```

**必须配置的项**:
- `FOFA_API_KEY`
- `FOFA_EMAIL`
- `HUNTER_API_KEY`

**可选配置的项**:
- 邮件通知配置
- Webhook 配置
- 存储配置

### 3. 启动服务

```bash
# 构建并启动所有服务 (后台运行)
docker-compose up -d --build

# 查看构建和启动日志
docker-compose logs -f

# 等待所有服务启动完成 (约 1-3 分钟)
```

### 4. 检查服务状态

```bash
# 查看所有容器状态
docker-compose ps

# 预期输出:
# NAME                COMMAND                  SERVICE             STATUS              PORTS
# unimap-mysql        "docker-entrypoint.s…"   mysql               running             0.0.0.0:3306->3306/tcp
# unimap-redis        "redis-server /usr/l…"   redis               running             0.0.0.0:6379->6379/tcp
# unimap-minio        "server /data --cons…"   minio               running             0.0.0.0:9000-9001->9000-9001/tcp
# unimap-api          "./unimap-api"           unimap-api          running             0.0.0.0:8080->8080/tcp
# icp-scheduler       "./icp-hunter schedu…"   icp-scheduler       running
# icp-worker-1        "./icp-hunter worker"    icp-worker-1        running
# icp-worker-2        "./icp-hunter worker"    icp-worker-2        running
# unimap-grafana      "/run.sh"                grafana             running             0.0.0.0:3000->3000/tcp
```

## 配置说明

### 环境变量配置 (.env)

#### 必需配置

```bash
# FOFA API 配置
FOFA_API_KEY=your_actual_fofa_api_key
FOFA_EMAIL=your_fofa_email@example.com

# Hunter API 配置
HUNTER_API_KEY=your_actual_hunter_api_key
```

#### 可选配置 - 通知系统

```bash
# 邮件通知 (用于每日报告)
EMAIL_SMTP_HOST=smtp.gmail.com
EMAIL_SMTP_PORT=587
EMAIL_USERNAME=your_email@gmail.com
EMAIL_PASSWORD=your_app_password
EMAIL_TO=admin@example.com

# Webhook 通知 (实时告警)
WEBHOOK_URL=https://oapi.dingtalk.com/robot/send?access_token=xxx
WEBHOOK_TYPE=dingtalk  # dingtalk / wechat / lark
```

#### 可选配置 - 存储系统

```bash
# 截图存储 (MinIO 默认已配置)
STORAGE_TYPE=minio
STORAGE_ENDPOINT=http://minio:9000
STORAGE_ACCESS_KEY=minioadmin
STORAGE_SECRET_KEY=minioadmin
STORAGE_BUCKET=icp-screenshots
```

#### 可选配置 - 系统参数

```bash
# 扫描策略 (逗号分隔)
SCAN_POLICIES=http_80,https_443,common_alt_ports

# Cron 调度 (每天凌晨2点)
CRON_SCHEDULE=0 2 * * *

# 最大并发数
MAX_CONCURRENCY=20

# 日志级别
LOG_LEVEL=info
```

### 数据库配置

数据库配置在 `docker-compose.yml` 中，如需修改：

```yaml
services:
  mysql:
    environment:
      MYSQL_ROOT_PASSWORD: your_root_password  # 修改 root 密码
      MYSQL_DATABASE: unimap_icp_hunter
      MYSQL_USER: unimap_user                   # 修改用户名
      MYSQL_PASSWORD: unimap_password           # 修改密码
```

修改后需要重建数据库：

```bash
# 停止并删除旧容器和数据
docker-compose down -v

# 重新启动
docker-compose up -d
```

### Redis 配置

Redis 配置文件位于 `configs/redis.conf`，主要配置：

```conf
# 内存限制 (2GB)
maxmemory 2gb
maxmemory-policy allkeys-lru

# 持久化策略
save 900 1
save 300 10
save 60 10000

# AOF 持久化
appendonly yes
appendfsync everysec

# 最大连接数
maxclients 10000
```

## 服务启动

### 启动所有服务

```bash
# 后台启动所有服务
docker-compose up -d

# 查看启动日志
docker-compose logs -f

# 等待服务就绪 (约 1-3 分钟)
```

### 启动特定服务

```bash
# 只启动数据库和 Redis
docker-compose up -d mysql redis minio

# 启动 UniMap API
docker-compose up -d unimap-api

# 启动 ICP-Hunter 服务
docker-compose up -d icp-scheduler icp-worker-1 icp-worker-2
```

### 重启服务

```bash
# 重启所有服务
docker-compose restart

# 重启特定服务
docker-compose restart icp-worker-1

# 重启并重建某个服务
docker-compose up -d --build icp-scheduler
```

### 停止服务

```bash
# 停止所有服务
docker-compose stop

# 停止并删除容器
docker-compose down

# 停止并删除容器和数据卷 (谨慎使用!)
docker-compose down -v
```

## 日常运维

### 1. 监控服务状态

```bash
# 查看容器运行状态
docker-compose ps

# 查看资源使用
docker stats

# 查看特定容器日志
docker-compose logs -f --tail=100 icp-worker-1
```

### 2. 数据库维护

```bash
# 进入 MySQL 容器
docker exec -it unimap-mysql mysql -u unimap_user -p

# 优化表 (定期执行)
OPTIMIZE TABLE assets;
OPTIMIZE TABLE icp_checks;

# 查看数据库大小
SELECT
    table_name AS "Table",
    ROUND(((data_length + index_length) / 1024 / 1024), 2) AS "Size (MB)"
FROM information_schema.TABLES
WHERE table_schema = "unimap_icp_hunter"
ORDER BY (data_length + index_length) DESC;
```

### 3. 备份和恢复

```bash
# 备份数据库
docker exec unimap-mysql mysqldump -u root -p unimap_icp_hunter > backup_$(date +%Y%m%d).sql

# 恢复数据库
docker exec -i unimap-mysql mysql -u root -p unimap_icp_hunter < backup_20260114.sql

# 备份 Redis 数据 (如果需要)
docker exec unimap-redis redis-cli BGSAVE
docker cp unimap-redis:/data/dump.rdb ./redis_backup_$(date +%Y%m%d).rdb
```

### 4. 日志管理

```bash
# 查看最近 100 行日志

# 查看特定时间范围的日志
docker-compose logs --since="2026-01-14T00:00:00" --until="2026-01-14T23:59:59"

# 清理旧日志 (防止磁盘占满)
docker system prune -f
```

```bash
git pull origin main

docker-compose up -d --build
```


## 故障排查

### 问题 1: 容器启动失败

```bash
# 查看具体错误
docker-compose logs [service_name]

# 常见原因:
# 1. 端口被占用
#    检查: netstat -tuln | grep 3306
#    解决: 修改 docker-compose.yml 中的端口映射

# 2. 环境变量缺失
#    检查: cat .env
#    解决: 填入必需的 API Keys

# 3. 磁盘空间不足
#    检查: df -h
#    解决: 清理磁盘或扩容
```

### 问题 2: 数据库连接失败

```bash
# 检查 MySQL 是否运行
docker ps | grep mysql

# 检查 MySQL 日志
docker logs unimap-mysql

# 手动连接测试
docker exec -it unimap-mysql mysql -u unimap_user -p

# 检查环境变量配置
echo $UNIMAP_DATABASE_HOST
echo $UNIMAP_DATABASE_PASSWORD
```

### 问题 3: API 调用失败

```bash
# 检查 API Key 是否正确
echo $FOFA_API_KEY

# 测试 API 连接性
curl -u "$FOFA_EMAIL:$FOFA_API_KEY" "https://fofa.info/api/v1/search/all?q=base64_query"

# 查看 API 服务日志
docker-compose logs unimap-api

# 检查网络连接
docker exec unimap-api ping -c 3 fofa.info
```

### 问题 4: Worker 不处理任务

```bash
# 检查 Redis 连接
docker exec -it icp-worker-1 redis-cli ping

# 检查任务队列长度
docker exec -it icp-worker-1 redis-cli XLEN icp:queue:probe

# 查看 Worker 日志
docker-compose logs icp-worker-1

# 检查调度器是否推送任务
docker-compose logs icp-scheduler
```

### 问题 5: 截图失败

```bash
# 检查 MinIO 是否运行
docker ps | grep minio

# 访问 MinIO 控制台
# http://localhost:9001 (minioadmin/minioadmin)

# 检查存储桶是否存在
docker exec -it unimap-minio mc ls

# 检查存储空间
docker exec -it unimap-minio mc df
```

### 问题 6: 内存不足

```bash

docker stats

# 编辑 configs/config.yaml
# 修改 max_concurrency: 10
```

### 问题 7: 任务堆积


## 性能优化建议

### 1. 数据库优化

- 定期执行 `OPTIMIZE TABLE`
- 添加必要的索引
- 调整 InnoDB 缓冲池大小
- 使用 SSD 存储

### 2. Redis 优化

- 调整 `maxmemory-policy` 为 `allkeys-lru`
- 合理设置过期时间
- 监控内存使用情况
- 使用持久化策略

### 3. Worker 扩展

- 根据负载调整 Worker 数量
- 调整 `max_concurrency` 参数
- 监控队列长度
- 避免过度并发

### 4. 网络优化

- 使用国内镜像源
- 配置 HTTP 代理
- 调整超时时间
- 启用连接池

### 5. 存储优化

- 定期清理旧截图
- 使用对象存储 CDN
- 压缩图片
- 设置生命周期策略

## 安全建议

### 1. 修改默认密码

```bash
# MySQL root 密码
# 编辑 docker-compose.yml
MYSQL_ROOT_PASSWORD=your_strong_password

# MySQL 用户密码
MYSQL_PASSWORD=your_strong_password

# Grafana 密码
GF_SECURITY_ADMIN_PASSWORD=your_strong_password

# MinIO 密码
MINIO_ROOT_PASSWORD=your_strong_password
```

### 2. 限制访问

```bash
# 只暴露必要端口
# 编辑 docker-compose.yml
# MySQL: 127.0.0.1:3306:3306 (只允许本地访问)
# Redis: 127.0.0.1:6379:6379 (只允许本地访问)
```

### 3. API Keys 管理

- 不要提交到 Git
- 使用密钥管理服务
- 定期轮换
- 最小权限原则

### 4. 日志安全

- 定期清理日志
- 不要记录敏感信息
- 设置日志级别
- 监控异常访问
