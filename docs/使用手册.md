### 数据流

```
1. 用户输入 UQL 查询
   ↓
2. UniMap 解析器解析 UQL
   ↓
3. 引擎编排器分发到多个引擎 (FOFA/Hunter/...)
   ↓
4. 结果合并器去重和补全
   ↓
5. 资产存入 MySQL
   ↓
6. 调度器生成扫描任务
   ↓
7. 任务推送到 Redis Stream
   ↓
8. Worker 消费任务并探测
   ↓
9. 分析器检测 ICP 备案
   ↓
10. 结果存入 MySQL
    ↓
11. 通知系统发送报告
```

## CLI 工具使用

### UniMap CLI

#### 基本语法

```bash
./unimap-api [command] [flags]
```

#### 常用命令

**1. 查询资产**

```bash
# 基本查询
./unimap-api query 'country="CN" && port="80" && protocol="http"'

# 指定引擎
./unimap-api query 'country="CN" && port="80"' --engines fofa,hunter

# 限制结果数量
./unimap-api query 'country="CN" && port="80"' --limit 100

# 输出为 JSON
./unimap-api query 'country="CN" && port="80"' --format json

# 保存到文件
./unimap-api query 'country="CN" && port="80"' --output results.json
```

**2. 验证 UQL 语法**

```bash
./unimap-api validate 'country="CN" && port="80" && protocol="http"'
# 输出: ✓ Valid UQL syntax
```

**3. 列出支持的引擎**

```bash
./unimap-api list-engines
# 输出:
# - fofa (enabled)
# - hunter (enabled)
# - zooomeye (disabled)
# - quake (disabled)
```

**4. 查看帮助**

```bash
./unimap-api --help
./unimap-api query --help
```

### ICP-Hunter CLI

#### 基本语法

```bash
./icp-hunter [command] [flags]
```

#### 常用命令

**1. 运行完整流程**

```bash
# 使用默认策略
./icp-hunter run --policy http_80

# 指定多个策略
./icp-hunter run --policy http_80,https_443

# 自定义 UQL (不使用预定义策略)
./icp-hunter run --uql 'country="CN" && port="80"' --engines fofa
```

**2. 手动触发扫描**

```bash
# 使用策略名称
./icp-hunter scan --policy https_443

# 指定任务参数
./icp-hunter scan --policy http_80 --max-records 1000 --page-size 50
```

**3. 启动调度器**

```bash
# 前台运行
./icp-hunter scheduler

# 后台运行 (使用 nohup)
nohup ./icp-hunter scheduler > scheduler.log 2>&1 &
```

**4. 启动工作节点**

```bash
# 前台运行
./icp-hunter worker

# 指定并发数
./icp-hunter worker --concurrency 10

# 后台运行
nohup ./icp-hunter worker > worker.log 2>&1 &
```

**5. 查看统计**

```bash
# 查看今日统计
./icp-hunter stats --today

# 查看最近 7 天统计
./icp-hunter stats --days 7

# 查看指定日期范围
./icp-hunter stats --from 2026-01-01 --to 2026-01-14

# 输出为 JSON
./icp-hunter stats --days 7 --format json
```

**6. 生成报告**

```bash
# 生成每日报告
./icp-hunter report --type daily --date 2026-01-14

# 生成周报告
./icp-hunter report --type weekly --date 2026-01-14

# 生成月报告
./icp-hunter report --type monthly --date 2026-01-14

# 保存到文件
./icp-hunter report --type daily --date 2026-01-14 --output report.html
```

**7. 白名单管理**

```bash
# 列出白名单
./icp-hunter whitelist list

# 添加白名单
./icp-hunter whitelist add --type domain --value "example.com" --reason "测试域名"
./icp-hunter whitelist add --type ip --value "192.168.1.1" --reason "内网IP"

# 删除白名单
./icp-hunter whitelist remove --id 1

# 批量导入
./icp-hunter whitelist import --file whitelist.txt
```

**8. 查看帮助**

```bash
./icp-hunter --help
./icp-hunter scan --help
./icp-hunter worker --help
```

## UQL 查询语言

### 基本语法

#### 1. 等于操作

```
country="CN"
port="80"
protocol="http"
```

#### 2. 不等于操作

```
country!="US"
port!="443"
```

#### 3. 模糊匹配

```
title~="管理后台"
body~="ICP备案"
server~="nginx"
```

#### 4. 正则匹配

```
body~="icp.*\\d+"
url~="https://.*\\.com"
```

#### 5. 范围查询

```
port IN ["80", "443", "8080", "8443"]
country IN ["CN", "HK", "TW"]
```

#### 6. 逻辑组合

```
# AND (与)
country="CN" && port="80" && protocol="http"

# OR (或)
country="CN" || country="HK"

# NOT (非)
!(country="US")

# 复杂组合
(country="CN" || country="HK") && port IN ["80", "443"] && !(title~="test")
```

#### 7. 范围查询

```
# 数值范围
port >= 80 && port <= 443

# 时间范围 (如果字段支持)
last_seen_at >= "2026-01-01"
```

### 支持的字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `ip` | 字符串 | IP地址 | `ip="192.168.1.1"` |
| `port` | 数字 | 端口 | `port="80"` |
| `protocol` | 字符串 | 协议 | `protocol="http"` |
| `country` | 字符串 | 国家代码 | `country="CN"` |
| `region` | 字符串 | 地区 | `region="beijing"` |
| `city` | 字符串 | 城市 | `city="shanghai"` |
| `title` | 字符串 | 网页标题 | `title~="登录"` |
| `body` | 字符串 | 网页内容 | `body~="ICP"` |
| `server` | 字符串 | Server头 | `server="nginx"` |
| `host` | 字符串 | 域名 | `host="example.com"` |
| `url` | 字符串 | 完整URL | `url~="https://"` |
| `asn` | 字符串 | ASN编号 | `asn="AS12345"` |
| `org` | 字符串 | 组织 | `org="Tencent"` |
| `isp` | 字符串 | 运营商 | `isp="China Telecom"` |

### 引擎特定字段

#### FOFA 支持的字段

```
ip, port, protocol, country, region, city,
title, server, domain, header, body,
cert, icon_hash, link, js_hash, css_hash
```

#### Hunter 支持的字段

```
ip, port, protocol, country, region, city,
title, server, domain, header, body,
cert, icon, link, component, os, app
```

### 实际使用示例

#### 示例 1: 查询中国的 HTTP 服务

```
country="CN" && port="80" && protocol="http"
```

#### 示例 2: 查询管理后台

```
country="CN" && (title~="管理" || title~="admin" || title~="登录") && port IN ["80", "443"]
```

#### 示例 3: 查询特定组件

```
body~="WordPress" && country="CN" && port="80"
```

#### 示例 4: 排除特定域名

```
country="CN" && port="80" && !(domain~="gov.cn") && !(domain~="edu.cn")
```

#### 示例 5: 复杂查询

```
(country="CN" || country="HK")
&& port IN ["80", "443", "8080", "8443"]
&& (title~="后台" || title~="管理" || body~="login")
&& !(domain~="test.com")
```

## 白名单管理

### 白名单类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `domain` | 域名后缀 | `gov.cn`, `edu.cn` |
| `domain_exact` | 完整域名 | `www.gov.cn` |
| `ip` | IP地址 | `192.168.1.1` |
| `cidr` | IP段 | `10.0.0.0/8` |
| `url` | URL前缀 | `https://internal.company.com` |

### 操作命令

#### 1. 添加白名单

```bash
# 添加域名后缀
./icp-hunter whitelist add --type domain --value "gov.cn" --reason "政府域名"

# 添加完整域名
./icp-hunter whitelist add --type domain_exact --value "www.example.com" --reason "特殊业务"

# 添加 IP
./icp-hunter whitelist add --type ip --value "192.168.1.100" --reason "内网服务器"

# 添加 IP 段
./icp-hunter whitelist add --type cidr --value "10.0.0.0/8" --reason "内网网段"

# 添加 URL 前缀
./icp-hunter whitelist add --type url --value "https://api.internal.com" --reason "内部API"
```

#### 2. 查看白名单

```bash
# 列出所有
./icp-hunter whitelist list

# 按类型筛选
./icp-hunter whitelist list --type domain

# 搜索
./icp-hunter whitelist list --search "gov"

# 输出为 JSON
./icp-hunter whitelist list --format json
```

#### 3. 删除白名单

```bash
# 按 ID 删除
./icp-hunter whitelist remove --id 1

# 批量删除
./icp-hunter whitelist remove --ids "1,2,3"
```

#### 4. 导入/导出

```bash
# 导入
./icp-hunter whitelist import --file whitelist.txt

# 导出
./icp-hunter whitelist export --file whitelist_backup.txt
```

### 白名单文件格式

```
# 格式: 类型,值,原因
domain,gov.cn,政府域名
domain,edu.cn,教育域名
domain,mil.cn,军事域名
ip,192.168.1.1,测试服务器
cidr,10.0.0.0/8,内网网段
url,https://internal.company.com,内部系统
```

### 白名单生效规则

1. **域名后缀匹配**: `domain="gov.cn"` 匹配 `www.gov.cn`, `api.gov.cn` 等
2. **精确匹配**: `domain_exact="www.gov.cn"` 只匹配 `www.gov.cn`
3. **IP 匹配**: `ip="192.168.1.1"` 精确匹配
4. **CIDR 匹配**: `cidr="10.0.0.0/8"` 匹配整个网段
5. **URL 前缀**: `url="https://api"` 匹配所有以该前缀开头的 URL

```bash
JOIN icp_checks c ON a.id = c.asset_id

INTO OUTFILE '/tmp/results.csv'
FIELDS TERMINATED BY ','
ENCLOSED BY '\"'
LINES TERMINATED BY '\n';"
```

### API 端点

## 最佳实践

### 1. 扫描策略建议

#### 策略 1: 基础扫描

```yaml
name: base_scan
uql: 'country="CN" && port IN ["80", "443"]'
engines: ["fofa", "hunter"]
page_size: 100
max_records: 5000
ports: [80, 443]
```

**适用场景**: 日常监控，覆盖最常见端口

#### 策略 2: 扩展扫描

```yaml
name: extended_scan
uql: 'country="CN" && port IN ["80", "443", "8080", "8443", "8000", "9000"]'
engines: ["fofa", "hunter"]
page_size: 50
max_records: 10000
ports: [80, 443, 8080, 8443, 8000, 9000]
```

**适用场景**: 每周深度扫描，覆盖常见替代端口

#### 策略 3: 重点扫描

```yaml
name: focused_scan
uql: 'country="CN" && (title~="管理" || title~="admin" || title~="登录" || body~="login") && port IN ["80", "443"]'
engines: ["fofa", "hunter"]
page_size: 50
max_records: 2000
ports: [80, 443]
```

**适用场景**: 重点目标扫描，关注管理后台

```bash
# 1. 停止服务
docker-compose stop

# 2. 恢复备份
docker exec -i unimap-mysql mysql -u root -p unimap_icp_hunter < backup.sql

# 3. 重启服务
docker-compose up -d
```

```bash
# 1. 更新 .env 文件
vim .env

# 2. 重启相关服务
docker-compose restart icp-scheduler icp-worker-1 icp-worker-2

# 3. 验证新 Key
docker exec -it icp-scheduler sh -c "echo \$FOFA_API_KEY"
```

## 常见问题 FAQ

### Q1: 如何添加新的搜索引擎支持?

**A**:
1. 在 `internal/adapter/` 创建新适配器
2. 实现 `EngineAdapter` 接口
3. 在 `orchestrator.go` 注册引擎
4. 更新 `configs/config.yaml`

### Q2: 扫描速度太慢怎么办?

**A**:
1. 增加 Worker 数量
2. 提高并发数配置
3. 使用多个 API Key
4. 优化 UQL 查询，减少结果集

### Q3: 如何排除某些 IP/域名?

**A**: 使用白名单功能
```bash
./icp-hunter whitelist add --type domain --value "example.com" --reason "排除原因"
```

### Q4: 如何查看详细的检测过程?

**A**:
1. 查看 Worker 日志: `docker-compose logs -f icp-worker-1`
2. 设置日志级别为 debug
3. 查看 Redis Stream 中的任务

### Q5: 如何处理 API 限流?

**A**:
1. 降低并发数
2. 增加请求间隔
3. 使用多个 API Key 轮询
4. 调整调度时间

### Q6: 数据库空间不足怎么办?

**A**:
1. 清理旧数据: `DELETE FROM icp_checks WHERE check_time < DATE_SUB(NOW(), INTERVAL 90 DAY);`
2. 优化表: `OPTIMIZE TABLE`
3. 备份并归档旧数据
4. 扩大磁盘空间

### Q7: 如何自定义 ICP 检测规则?

**A**: 编辑 `configs/config.yaml` 中的 `icp.patterns`，添加新的正则表达式

### Q8: 如何发送通知到多个渠道?

**A**: 同时配置邮件和 Webhook，系统会自动发送到所有启用的渠道

### Q9: 如何重试失败的任务?

**A**: Worker 会自动重试 3 次，如果仍失败，可以:
1. 查看失败原因
2. 手动重新触发扫描
3. 检查网络和 API Key

### Q10: 如何监控系统健康状态?

**A**:
1. 使用 Grafana 仪表盘
2. 查看 `docker-compose stats`
3. 监控 Redis 队列长度
4. 检查日志中的错误
