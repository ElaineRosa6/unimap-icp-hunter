## 第二部分：现有业务能力清单

以下为当前项目中已实现的所有业务功能，标记了哪些适合作为定时任务候选。

### 一、多引擎搜索查询

| # | 功能 | 说明 | 定时任务候选 |
|---|------|------|:-----------:|
| Q1 | **UQL 统一查询** | 解析 UQL 语法，多引擎并行搜索，合并结果 | **是** |
| Q2 | **浏览器异步查询** | 在真实浏览器中打开搜索结果，支持分页自动加载 | **是** |
| Q3 | **查询结果导出** | 将查询结果导出为 JSON 或 Excel 格式 | **是** |
| Q4 | **引擎配额查询** | 查询各搜索引擎的 API 剩余配额 | **是** |
| Q5 | **查询缓存管理** | 内存/Redis 缓存，动态 TTL 调整 | 否（内部自动） |

### 二、截图系统

| # | 功能 | 说明 | 定时任务候选 |
|---|------|------|:-----------:|
| S1 | **单 URL 截图** | 对指定 URL 进行截图（CDP / Extension 双模式） | **是** |

| S3 | **批量 URL 截图** | 上传 URL 列表，批量并发截图 | **是** |
| S4 | **批量引擎+目标截图** | 先搜索引擎搜索，再对结果中的目标截图 | **是** |

```
┌─────────────────────────────────────────────────┐
│                 Web UI (前端)                    │
│  ┌───────────┐ ┌───────────┐ ┌───────────────┐  │
│  │ 任务创建   │ │ 任务列表   │ │ 执行历史/日志  │  │
│  │ & 编辑表单 │ │ & 启停控制 │ │ & 结果查看    │  │
│  └───────────┘ └───────────┐ └───────────────┘  │
└────────────────────────────┼─────────────────────┘
                             │
┌────────────────────────────┼─────────────────────┐
│              Task Scheduler (调度器)              │
│  ┌──────────────┐  ┌───────┴───────┐             │
│  │ Cron Parser  │  │  任务注册表    │             │
│  │ (gocron)     │  │  (持久化存储) │             │
│  └──────────────┘  └───────┬───────┘             │
│                            │                      │
│  ┌─────────────────────────┴───────────────────┐  │
│  │           Task Executor (执行器)             │  │
│  │  ┌────────┐ ┌──────────┐ ┌──────────────┐   │  │
│  │  │ Query  │ │Screenshot│ │ Tamper Check  │   │  │
│  │  │ Runner │ │ Runner   │ │ Runner        │   │  │
│  │  └────────┘ └──────────┘ └──────────────┘   │  │
│  │  ┌────────┐ ┌──────────┐ ┌──────────────┐   │  │
│  │  │Monitor │ │ Cookie   │ │ Export       │   │  │
│  │  │ Runner │ │ Verifier │ │ Runner       │   │  │
│  │  └────────┘ └──────────┘ └──────────────┘   │  │
│  └──────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

### 建议依赖

- **Cron 库**: `github.com/robfig/cron/v3`（支持秒级精度、秒域可选）
- **持久化**: 现有快照存储目录扩展或 SQLite
- **执行控制**: 互斥锁（同任务不并发）、超时控制、重试策略
- **结果存储**: 执行历史写入本地存储，可通过 API 查询
- **前端**: 在现有 Web UI 中新增"定时任务"页面
